# エンジニアリング原則（Core Engineering Principles）

このファイルには、プロジェクト全体に適用されるエンジニアリング原則を記載します。

## 思考と成熟

- 目先の成果で自分を証明するのではなく、チームに長期的な理解を残すという視点
- 仕様をコードだけで語るのではなく、価値を言葉で確かめてから手を動かすという視点
- 技術的な正しさで完結するのではなく、人との信頼がソフトウェアを支えるという視点
- 自分の手数を増やすのではなく、必要な判断を減らす成熟を求めるという視点

## シンプルな実装

- 巧妙なトリックではなく、半年後の自分が理解できる構造を優先するという視点
- 想定外の未来へ備えるより、目の前の価値に必要な最小機能へ集中するという視点
- 設計を積み上げるより、不要な要素を削って本質を残すという視点
- 抽象の層を増やすのではなく、目的を一行で説明できる実装を意識するという視点

## 安定した基盤

- トレンドのフレームワークを追いかけるのではなく、チームが保守し続けられる技術を選ぶという視点
- 派手な指標を追うより、合意したパフォーマンス要件を着実に満たすという視点
- 個人の勘頼みの運用ではなく、復旧時間目標を守れる標準手法を全員で磨くという視点
- 暗黙の手順に頼るより、自動化と検証で再現性を担保するという視点

## プロセスとコミュニケーション

- バグが出てから修正を競うのではなく、誤解を対話で先に潰すという視点
- 各自の流儀を貫くのではなく、規約や型チェックを仲間を守る防御線と捉えるという視点
- プロセスの儀式を守ること自体ではなく、目的に合わせて手段を調節する柔軟さを持つという視点
- 聞かれるまで黙っているより、初期の違和感を早めに共有して認識を揃えるという視点

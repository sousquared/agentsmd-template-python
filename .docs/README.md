# 知識ベース（Knowledge Base）

このディレクトリには、AI エージェントと人間の両方が参照する知識をモジュール化して格納しています。

**更新責任者**: プロジェクトメンテナー

## カテゴリ構成

知識は以下の 5 つのカテゴリに分類されています：

- **[00-core-principles.md](00-core-principles.md)**: エンジニアリング原則（常に参照）
- **[01-coding-standards/](01-coding-standards/)**: コーディング規約
- **[02-workflow/](02-workflow/)**: ワークフロー・プロセス
- **[03-domain-specific/](03-domain-specific/)**: ドメイン固有の知識
- **[04-reference/](04-reference/)**: リファレンス情報

各カテゴリの詳細なファイル一覧は、各カテゴリの README を参照してください。

## 使用方法

エージェントは、作業内容に応じて以下のカテゴリを参照してください：

| 作業内容                     | 参照カテゴリ            | 優先度 |
| ---------------------------- | ----------------------- | ------ |
| すべての作業                 | `00-core-principles.md` | 最高   |
| コード実装                   | `01-coding-standards/`  | 高     |
| PR 作成、Git 操作            | `02-workflow/`          | 高     |
| ドメイン実装                 | `03-domain-specific/`   | 中     |
| セットアップ、ライブラリ使用 | `04-reference/`         | 低     |

## ファイル追加のガイドライン

1. **単一責任**: 1 ファイル 1 トピックを原則とする
2. **自己完結**: 各ファイルは独立して理解できる内容にする
3. **相互参照**: 必要に応じて他のファイルへの参照を記載

## AGENTS.md との関係

- **AGENTS.md**: エントリーポイント。最小限の概要と、この知識ベースへの参照のみを記載
- **.docs/**: 詳細な知識をモジュール化して格納

この構造により、エージェントは必要な知識のみを参照でき、コンテキストの肥大化を防ぎます。

## 更新ルール

各カテゴリの更新ルールは、各カテゴリの README を参照してください。変更履歴は Git ログで確認できます。
